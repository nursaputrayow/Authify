Introduction

Base URL: http://authify.test

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

POST api/v1/register

Example request:
curl --request POST \
    "http://authify.test/api/v1/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"name\": \"John Doe\",
    \"phone\": \"+1234567890\",
    \"email\": \"john.doe@example.com\",
    \"password\": \"password123\"
}"

const url = new URL(
    "http://authify.test/api/v1/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "name": "John Doe",
    "phone": "+1234567890",
    "email": "john.doe@example.com",
    "password": "password123"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

POST api/v1/register

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

name   string   

The name of the user. Must be at least 2 characters. Must not be greater than 50 characters. Example: John Doe

phone   string   

The phone number of the user in international format. Must match the regex /^+\d{10,15}$/. Example: +1234567890

email   string   

The email address of the user. Must be a valid email address. Example: john.doe@example.com

password   string   

The password for the user account. Must be at least 6 characters. Example: password123
